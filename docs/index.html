chatjimmy-proxy points Claude Code at ChatJimmy's Llama 3.1 8B — a free model that decodes at ~14,500 tokens/sec. You keep the full agentic loop; the model behind it just got ~150× faster.

  1. 1.0The idea.

    Claude Code is one of the best agentic coding harnesses there is — but the planning loop, tool use, permissions and sub-agents are all harness, not model. ChatJimmy serves Llama 3.1 8B at an almost absurd speed: ~14,500 tokens/sec, ~1.1 ms to first token. That's roughly 10× faster than Groq and 150–280× faster than frontier models.

    It has no official API and no tool-calling. chatjimmy-proxy bridges the gap — a tiny local server that speaks the Anthropic Messages API on one side and ChatJimmy's raw text endpoint on the other, translating tool calls in between.

    Speed buys reliability.

    Each inference is ~3 ms, so the proxy can draw the model 5× per turn to land a valid tool call — and the whole turn is still faster than a single frontier-model call. The speed is what makes a weak 8B usable as a coding agent.

  2. 2.0How it works.

    Claude Code talks to the proxy (via deep-claude, which isolates your real Anthropic login). The proxy does the work that turns a fast-but-weak model into a coding agent:

    1. swaps Claude Code's giant system prompt for a compact, tuned one,
    2. injects a <tool_call> format and parses the model's text back into Anthropic tool_use blocks,
    3. best-of-N: re-samples until a valid tool call parses,
    4. grounded best-of-N: picks the answer most supported by actual tool output, and
    5. guards against destructive shell commands.

    Backend-switchable.

    Set BACKEND=openrouter to route to meta-llama/llama-3.1-8b-instruct instead — the same model on a billed API, for tuning without hammering ChatJimmy.

  3. 3.0The setup.

    You'll need Claude Code, deep-claude, and Node 18+. Clone, build, and register ChatJimmy as a deep-claude endpoint:

    sh 
    git clone https://github.com/dennisonbertram/chatjimmy-proxy
cd chatjimmy-proxy
npm install && npm run build

# register chatjimmy as a deep-claude endpoint (one time)
deep-claude endpoints add chatjimmy http://localhost:3000

  4. 4.0The run.

    One launcher does everything — it auto-starts the proxy and opens a clean Claude Code session on ChatJimmy. Run it from the project directory you want to work in (Claude Code sandboxes file tools to the working directory).

    console 
    $ cd your-project
$ chatjimmy-code
…a clean Claude Code session, on Llama 3.1 8B @ ~14,500 tok/s…

$ chatjimmy-code -p "create an index.html with a Hello World heading"
…done in ~1.5s…

    The launcher bundles the settings that make the weak model work: compact prompt, no MCP servers, no skills, coding-only toolset.

  5. 5.0The speed.

    Measured with ./eval/speed-benchmark.sh against ChatJimmy's own telemetry and real end-to-end runs:

    ~14,500 tok/s— decode rate
    vs ≈1,250 for Groq's Llama-8B, ≈50–100 for frontier models.
    ~1.1 ms
    time-to-first-token.
    ~8 ms
    of inference to generate a full HTML page (~1.5 s end-to-end including harness).
    ~330 ms
    model time per agent turn — including up to 5× best-of-N resampling.

  6. 6.0The quality & the guardrails.

    After tuning the prompt and harness, on the real Claude Code agent loop the core four operations — read, edit, create, grep — land 5/5. Harder multi-step and multi-file tasks reach ≈0.70 on ChatJimmy's quantized instance (1.0 on OpenRouter's). It's an 8B model: superb for focused file ops, weaker on tangled multi-file logic.

    Safe by default.

    A weak model can hallucinate commands, so the proxy refuses destructive or privileged shell calls (sudo, rm -rf /, curl…|sh, writes to /etc/*) and never forces a tool call — "hi" gets a plain reply, not an invented command. Run it on code you trust, like any agent.

The reference.

Override anything through the environment.

Run
chatjimmy-code [args…]Clean Claude Code session on ChatJimmy. Args pass through to claude.
chatjimmy-code -p "task"One-shot print mode.
./eval/real-path-eval.sh NScore the four core tasks, N runs each.
./eval/speed-benchmark.shDecode rate + end-to-end timing.
Configure (env)
BACKENDchatjimmy (default) or openrouter.
TOOL_SAMPLE_ATTEMPTSbest-of-N draws for a valid tool call (default 5).
ANSWER_SAMPLE_ATTEMPTSgrounded answer candidates (default 3).
CHATJIMMY_TOOLStools advertised to the model.
MAX_SYSTEM_BYTEStrim the prompt to ChatJimmy's ~24 KB ceiling.
